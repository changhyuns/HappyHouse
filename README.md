### 추가 기능

#### 1. 개요
집을 선택하는데 있어서 고려해야 할 요소는 많지만, 주택에서 발생하는 <br>
가장 큰 문제점을 우선적으로 해소하는 것이 중요하다고 판단하였기 때문에<br>
몇 가지 제한된 방향에서의 알고리즘 적용 가능한 분야를 고려하였다.<br>

본 알고리즘 추가 적용 기획서에서는 크게 __환경__ , __대중교통__ , __주차__ 
이렇게 세 가지 요소에 대해 평가하는 것이 중요하다고 결정하였다.  <br><br>


### 요소1 : 환경 (k-means 알고리즘) 
<br>

첫 번째로 고려할 요소는 __환경__ 이다. <br>
해당 지역에 '살기 좋은 정도'를 기준으로 판단하려고 한다.<br>
사용할 API 데이터는 아래와 같다.<br><br>

- 어린이보호구역 정보  
- 서울시 음주실태 통계  
- 서울시 흡연실태 통계  

<br>
어린이보호구역의 경우, 각종 유해시설이 입지할 수 없을 뿐만 아니라 차량속도등 다양한 위법행위에 대한 처벌도 강화된다. <br><br>

따라서 해당 주택 근처에 많은 어린이 보호구역이 있을 경우 상대적으로 좋은 환경이라고 판단하였고<br>
음주실태와 흡연실태의 경우, 주택에서 가까이 있을수록 유해한 요소로 판단하였다.<br><br>

따라서 어린이보호 구역은 +   음주/흡연 실태는 -  요소로 처리하며 <br>
해당 주택과의 거리를 K-means 알고리즘을 통해 산출한 점수로<br>

'살기 좋은 정도'를 판별할 수 있다. <br><br>


### 요소2 : 대중교통 (다익스트라 알고리즘)
<br>

두 번째로 고려할 요소는 __대중교통__ 이다. <br>
차량이 있는 경우에는 주차 문제가 심각하지만, 차량이 없는 경우 대중교통의 불편하다면 <br>
이동에 제약이 생기기 때문에 무척이나 중요한 요소라고 판단하였다.<br>
대중교통 점수를 산정하기 위해서 활용할 API 데이터는 아래와 같다. <br><br>

- 버스노선 기본 정보( 다익스트라 )  
- 노선별 지하철역 정보( 다익스트라 )   
- 노선별 정류장별 승하차인원 정보 ( 시간대별 )  
- 서울시 나눔카 예약 가능 차랑정보 ( 사용가능 차량 수 )  

<br>
기본적으로 버스 정류장과 지하철역이 얼마나 가까이 있는지는 거주지 선택에 있어 매우 중요한 요소이다.<br><br>

따라서, 주택과 버스 정류장 지하철역과의 거리를 다익스트라(최단경로)알고리즘을 통해 점수를 산출하고, <br>
승하차 인원에 따라 원할한 이용이 불가능할 수 있기 때문에 정류장별 승하차 인원 정보를 확인하여 점수에 합산한다. <br><br>

마지막으로, 서울시 나눔카라는 공유 자동차를 원할하게 이용할 수 있는 거주지의 경우 추가적인 점수를 부여한다. <br>
버스 승하차인원은 통상 30명을 기준으로 계산하여 30명이 넘어가면 점수에서 차감되도록 구성한다. <br>

 + 점수 = (500 - 버스정류장 거리(다익스트라) - (30 - 승하차인원정보) ) + (500 - 지하철역 거리) + (나눔카 이용가능 대수 * 10)<br>

### 요소3 : 주차 (거리 계산, 비교 등)
<br>

세 번째로 고려할 요소는 __주차__ 이다. <br>
자차를 보유한 사용자의 경우, 거주지 선택에 있어서 <br>
주차 관련 문제도 중요하게 고려할 대상이다. <br><br>

따라서 주차 여건을 확인하기 위해 활용할 API 데이터는 아래와 같다. <br><br>
  
- 공영주차장 현황 정보  
- 민간위탁주차장 현황 정보  
- 서울시 주차장 확보율 통계  
- 전기차충전소 정보  
- 거주자우선주차 정보  

<br>

데이터를 활용하여 해당 주택과의 거리, 주차비용, 가능대수를 종합적으로 고려하여 '주차 점수'를 산출한다.

+ 주택과의 거리는 300m 이내 ,  주차비용 발생은 -  
+ 아파트의 경우, 주차 가능대수는 1.3대를 기준으로 미달 시 -  <br>
